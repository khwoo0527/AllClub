name: 에이전트 절차를 따르면서 대화 맥락 활용
description: 에이전트가 담당하는 작업을 직접 수행할 때, 해당 에이전트의 구조화된 절차를 반드시 따르면서 대화 맥락을 활용한다.
type: feedback
---

에이전트가 설계된 작업(ROADMAP 생성, Sprint 계획, 코드 리뷰 등)을 대화 중에 직접 수행할 때, 해당 에이전트의 구조화된 절차를 반드시 따른다.

**Why:** 에이전트는 구조화된 절차, 템플릿, 품질 체크리스트를 강제하므로 빠뜨리는 단계가 없다. 대화 맥락만 믿고 자유롭게 작성하면 단계를 건너뛰거나 품질이 들쭉날쭉해진다.

**How to apply:**
1. 해당 작업의 에이전트 문서를 먼저 읽는다 (`.claude/agents/xxx.md`)
2. 에이전트의 **단계별 절차**를 순서대로 따른다
3. 에이전트의 **품질 검증 체크리스트**를 마지막에 반드시 수행한다
4. 에이전트의 **템플릿/출력 형식**을 그대로 사용한다
5. 여기에 **대화에서 얻은 맥락**(사용자 요구, 결정 사항, 뉘앙스)을 추가로 반영한다

이 원칙은 모든 에이전트 작업에 적용:
- prd-to-roadmap → ROADMAP 작성 시
- phase-planner → Phase 계획 시
- sprint-planner → Sprint 계획 시
- sprint-review → 코드 리뷰 시
- sprint-close → Sprint 마무리 시
- deploy-prod → 배포 준비 시
