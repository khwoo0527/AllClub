description: .claude 범용화 작업 시 바이브 코딩 결과물 품질이 떨어지면 안 됨. 기술 스택 전문가 수준 유지 필수.
type: feedback
---

범용화는 전문성 희석이 아니다. 어떤 기술 스택이든 해당 분야 시니어 개발자 수준의 전문가로 동작해야 한다.

**Why:** 사용자가 .claude 폴더를 범용 프레임워크로 만들고 있지만, 바이브 코딩 결과물의 품질이 떨어지는 것은 절대 허용하지 않음. 범용화의 목적은 "어떤 프로젝트에서든 최고 품질"이지 "평균적으로 무난한 품질"이 아님.

**How to apply:**
- 에이전트/규칙 문서에서 기술 스택 로드 시 "참고" 수준이 아닌 "해당 기술의 시니어 전문가로서 베스트 프랙티스를 반드시 적용"하는 톤으로 작성
- rules/{tech}.md는 해당 기술의 깊은 전문 지식을 담아야 함 (얕은 가이드라인 X)
- 코드 리뷰, 구현, 설계 모두 해당 기술 스택의 최고 수준 품질 기준 적용
- "범용적으로 작성" ≠ "두루뭉술하게 작성"
